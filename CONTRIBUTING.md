# Contribuindo para PulseDataLabs

Obrigado por considerar contribuir! Toda ajuda é bem-vinda.

## Como contribuir

### Reportando bugs
- Use o [issue tracker](https://github.com/PulseDataLabs/pulsedatalabs.github.io/issues)
- Descreva o problema, passos para reproduzir e ambiente (navegador, SO)

### Sugerindo melhorias
- Abra uma issue com a label `enhancement`
- Explique o caso de uso e por que seria útil

### Pull Requests
1. Fork o repositório
2. Crie uma branch: `git checkout -b feature/nome-da-feature`
3. Faça suas alterações
4. Commit: `git commit -m "feat: descrição clara"`
5. Push: `git push origin feature/nome-da-feature`
6. Abra um PR

## Padrões de commit
Use [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` nova funcionalidade
- `fix:` correção de bug
- `docs:` documentação
- `style:` formatação (sem mudança de lógica)
- `refactor:` refatoração
- `chore:` tarefas de manutenção

## Código de conduta
Ao participar, você concorda em seguir nosso [Código de Conduta](CODE_OF_CONDUCT.md).
